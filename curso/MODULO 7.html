Texto pré-formatado

O elemento <pre> preserva espaços e quebras de linha exatamente como foram escritos. É muito usado para mostrar exemplos e trechos de código.

Quando usar <pre>?

Se precisares que o browser respeite “espaços extra” e “linhas em branco”, usa <pre>. Para código, é comum combinar <pre> com <code>: <pre><code>...</code></pre>.

Atenção: como o texto aparece “monoespaçado” e preserva tudo, pode ficar largo. Mantém linhas curtas quando possível.

Exemplo de texto pré-formatado

Texto formatado (resultado)

PRIMEIRA LINHA
SEGUNDA LINHA
TERCEIRA LINHA

E S P A Ç O S   E X T R A

HTML

<pre>PRIMEIRA LINHA
SEGUNDA LINHA
TERCEIRA LINHA

E S P A Ç O S   E X T R A</pre>

Boas práticas

  • Para código, prefere <pre><code> (semântica melhor).
  • Para mostrar tags literalmente, lembra-te das entidades: &lt; e &gt;.
  • Se um bloco for muito largo, podes permitir scroll horizontal (o browser já faz isso em muitos casos).

Exemplo (resultado)

<h1>Olá</h1>
<p>Isto é um parágrafo.</p>

HTML (com entidades)

<pre><code>&lt;h1&gt;Olá&lt;/h1&gt;
&lt;p&gt;Isto é um parágrafo.&lt;/p&gt;</code></pre>

Exercício prático

Edita o HTML e clica em Processar para veres o resultado no preview. Experimenta adicionar linhas em branco e espaços no início de uma linha.

Dica: escreve uma “escadinha” com espaços no início das linhas e vê como o <pre> preserva tudo.
