Formatação de caracteres

Alguns estilos são semânticos (dizem “o que é” o texto) e outros são apenas visuais (dizem “como parece”). Em HTML5, dá-se preferência ao significado (semântica) e usa-se CSS para o aspeto.

Formatação semântica vs. formatação visual

  • Semântica (lógica): indica o papel do texto (ex.: <em> = ênfase, <strong> = importância).
  • Visual (física): altera a aparência (ex.: <b>, <i>), mas pode não comunicar significado.

Regra simples: sempre que houver um elemento semântico adequado, usa-o. A apresentação (cores, tamanhos, fontes) resolve-se com CSS.

Nota: <strike> é obsoleto. Em HTML5 usa-se <s> (semântica: “já não é relevante”) ou CSS (apresentação).

Descritores visuais (históricos)

Estes tags existem por compatibilidade, mas hoje é comum usar CSS para controlar o aspeto. Ainda assim, é útil conhecer:

Texto formatado (resultado)

Texto em negrito

Texto em itálico

Texto sublinhado

Subscrito: H2O

Sobrescrito: y = x3

HTML

Texto em <b>negrito</b>
Texto em <i>itálico</i>
Texto <u>sublinhado</u>
H<sub>2</sub>O
y = x<sup>3</sup>

Dica moderna: para “negrito com significado”, prefere <strong>; para “itálico com significado”, prefere <em>.

Entidades HTML (caracteres especiais)

Algumas letras/símbolos podem ser representados por entidades (útil quando precisas escrever caracteres reservados como < e >):

Resultado Entidade
< &lt;
> &gt;
© &copy;
& &amp;
" &quot;

Com <meta charset="utf-8"> podes escrever acentos diretamente. Entidades continuam úteis para símbolos reservados.

Exercício prático

Copia/edita o HTML e clica em Processar para veres o resultado no preview. Podes experimentar trocar tags semânticas (em, strong) por visuais (b, i) e comparar.

Dica: tenta escrever &lt;p&gt; para mostrar literalmente <p> no preview.
