Como foi criado o formulário de inscrição

Este documento explica passo a passo todas as técnicas, decisões de design e código utilizados para construir o formulário de inscrição em cursos de informática.

O formulário foi construído com tecnologias web padrão — HTML5, CSS3 e JavaScript puro — sem qualquer framework ou biblioteca externa. É um único ficheiro .html autónomo que funciona diretamente no browser.

📄

HTML5 semântico

Estrutura com elementos nativos de formulário.

🎨

CSS3 moderno

Variáveis CSS, Grid, Flexbox e animações.

⚡

JavaScript puro

📱

Responsivo

Adapta-se a ecrãs de desktop, tablet e mobile.

01

Estrutura HTML

O ficheiro começa com a declaração <!DOCTYPE html> e o atributo lang="pt" para indicar ao browser que o conteúdo está em português. O <head> carrega a fonte do Google Fonts e toda a folha de estilos CSS.

HTML — Estrutura base
<!DOCTYPE html>
<html lang="pt">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Inscrição em Curso de Formação</title>
  <link href="https://fonts.googleapis.com/css2?family=..." rel="stylesheet">
  <style> /* CSS aqui */ </style>
</head>
<body>
  <header class="page-header">...título...</header>
  <div class="card">
    <div class="card-header">...cabeçalho azul...</div>
    <div class="card-body">
      <!-- Barra de progresso -->
      <div class="steps">...</div>
      <form id="main-form" novalidate>
        <div id="step-1">...dados pessoais...</div>
        <div id="step-2" class="hidden">...curso...</div>
        <div id="step-3" class="hidden">...confirmação...</div>
      </form>
      <div id="success-screen">...ecrã final...</div>
    </div>
  </div>
  <script>/* JavaScript aqui */</script>
</body>
💡 O atributo novalidate no <form> desativa a validação nativa do browser, permitindo-nos criar mensagens de erro personalizadas com CSS e JavaScript.
02

CSS e Variáveis

Todo o sistema de cores e espaçamentos é definido com CSS Custom Properties (variáveis CSS) no seletor :root. Isto permite alterar a aparência do formulário inteiro mudando apenas um valor.

CSS — Variáveis de design
:root {
  /* Cores base */
  --bg:       #f4f6fb;   /* fundo da página */
  --surface:  #ffffff;   /* fundo do card */
  --primary:  #2563eb;   /* azul principal */
  --primary-d:#1d4ed8;   /* azul escuro (hover) */
  --primary-l:#eff6ff;   /* azul muito claro (fundo) */
  --success:  #16a34a;   /* verde */
  --danger:   #dc2626;   /* vermelho de erro */
  --text:     #111827;   /* texto principal */
  --muted:    #6b7280;   /* texto secundário */
  --border:   #e5e7eb;   /* bordas */

  /* Layout */
  --radius:   12px;
  --shadow:   0 4px 24px rgba(0,0,0,.08);
}

O layout do formulário usa CSS Grid para organizar campos lado a lado:

CSS — Grelha de campos
/* Grelha de 2 colunas para os campos */
.form-row {
  display: grid;
  grid-template-columns: 1fr 1fr;  /* 2 colunas iguais */
  gap: 1rem;
}

/* Classe para campo que ocupa as 2 colunas */
.form-full {
  grid-column: 1 / -1;  /* da coluna 1 até à última */
}

Tipografia e Paleta de Cor

A fonte usada é a Plus Jakarta Sans, carregada do Google Fonts. É uma fonte moderna, legível e adequada para formulários. A fonte monoespaçada DM Mono é usada para a referência de inscrição.

Elementos do Formulário

Cada campo segue sempre a mesma estrutura: um <div class="form-group"> que agrupa a etiqueta, o campo e a mensagem de erro.

HTML — Estrutura de um campo
<div class="form-group">
  <!-- Etiqueta com asterisco de obrigatório -->
  <label>

  </label>

  <!-- Campo de input -->
  <input type="email"
         id="email"
         placeholder="ex: luisa@email.com"
         autocomplete="email">

  <!-- Mensagem de erro — oculta por defeito -->
  <span class="field-error" id="err-email">
    Introduz um email válido.
  </span>

O estilo dos campos muda com :focus para dar feedback visual ao utilizador:

CSS — Estados dos campos

input, select, textarea {
  border: 1.5px solid var(--border);
  background: #f9fafb;
  transition: border-color .2s, box-shadow .2s;
}

/* Quando o campo está ativo / em foco */
input:focus {
  border-color: var(--primary);
  background: var(--surface);
  box-shadow: 0 0 0 3px rgba(37,99,235,.12);
}

/* Quando há erro de validação */
input.error {
  border-color: var(--danger);
  background: var(--danger-l);
}
✅ O atributo autocomplete nos campos de texto ajuda o browser a preencher automaticamente dados do utilizador, melhorando a usabilidade.

A lista pendente de cursos usa o elemento nativo <select> do HTML. Para organizar os 22 cursos em categorias, usa-se o elemento <optgroup> que cria grupos visuais dentro da lista.

  </optgroup>

  <!-- Grupo 2 -->

    <option value="sys-linux">Administração de Sistemas Linux</option>
    <!-- mais opções... -->
  </optgroup>

Quando o utilizador escolhe um curso, a função atualizarCurso() em JavaScript consulta um objeto com todos os dados e mostra um cartão informativo dinâmico:

JavaScript — Objeto de dados dos cursos
const cursos = {
  'web-fund': {
    titulo: 'Fundamentos de HTML, CSS e JavaScript',
    horas:  '60h',
    nivel:  'Iniciante',
    preco:  '290€',
    desc:   'Aprende as tecnologias base da web moderna...'
  },
  'web-php': { /* ... */ },
  // 22 cursos no total
};

function atualizarCurso() {
  const val  = document.getElementById('curso').value;
  const prev = document.getElementById('course-preview');

  if (!val || !cursos[val]) {
    prev.classList.remove('show');
    return;
  }
  const c = cursos[val];
  document.getElementById('prev-titulo').textContent = c.titulo;
  document.getElementById('prev-meta').innerHTML =
    `<span class="chip">${c.horas}</span>` +
    `<span class="chip green">${c.nivel}</span>` +
    `<span class="chip amber">💶 ${c.preco}</span>`;
  document.getElementById('prev-desc').textContent = c.desc;
  prev.classList.add('show');
}
💡 O cartão de pré-visualização do curso usa a classe CSS .course-preview com display: none por defeito. Quando o utilizador seleciona um curso, o JavaScript adiciona a classe .show que muda o display para flex.
06

Radio Cards — Seleção Visual

Os botões de seleção de turno e modalidade usam uma técnica chamada radio card: um <label> estilizado como cartão que envolve um <input type="radio"> invisível. Clicar no cartão ativa o radio button.

HTML — Radio card
<label class="radio-card"
       onclick="selectRadio(this,'turno')">

  <!-- Input oculto com display:none no CSS -->
  <input type="radio" name="turno" value="Manhã">

  <span class="rc-title">🌅 Manhã</span>
  <span class="rc-sub">09h00 – 13h00</span>
</label>
CSS — Estilo do radio card
.radio-card {
  border: 1.5px solid var(--border);
  border-radius: 8px;
  padding: .75rem 1rem;
  cursor: pointer;
  transition: all .15s;
}
.radio-card:hover {
  border-color: var(--primary);
  background: var(--primary-l);
}
/* Classe adicionada pelo JavaScript quando selecionado */
.radio-card.selected {
  border-color: var(--primary);
  background: var(--primary-l);
}
JavaScript — Selecionar radio card
function selectRadio(el, name) {
  // Remove a seleção de todos os cards com o mesmo name
  document.querySelectorAll(`[name="${name}"]`)
    .forEach(r => r.closest('.radio-card')
    .classList.remove('selected'));

  // Marca o card clicado como selecionado
  el.classList.add('selected');

  // Ativa o input radio dentro do card
  el.querySelector('input').checked = true;
}
07

Formulário Multi-Passo

O formulário divide-se em 3 passos. Cada passo é um <div> independente dentro do <form>. Apenas um passo fica visível de cada vez — os restantes têm a classe hidden que aplica display: none.

1

Dados Pessoais

Nome, apelido, email, telemóvel, data de nascimento, NIF, habilitações e situação profissional.

2

Escolha do Curso

Lista pendente de cursos, turno, modalidade, data de início e informações adicionais.

3

Revisão e Confirmação

Resumo de todos os dados inseridos, aceitação dos termos e botão de submissão final.

JavaScript — Avançar para o passo seguinte
function nextStep(from) {
  // Valida o passo atual antes de avançar
  if (from === 1 && !validateStep1()) return;
  if (from === 2 && !validateStep2()) return;
  if (from === 2) preencherResumo(); // preenche o passo 3

  // Esconde o passo atual
  document.getElementById('step-' + from)
    .classList.add('hidden');

  // Mostra o passo seguinte
  document.getElementById('step-' + (from+1))
    .classList.remove('hidden');

  // Atualiza a barra de progresso
  document.getElementById('step-ind-' + from)
    .classList.replace('active', 'done');
  document.getElementById('step-ind-' + (from+1))
    .classList.add('active');

  // Faz scroll para o topo
  window.scrollTo({ top: 0, behavior: 'smooth' });
}
08

Validação dos Campos

A validação é feita em JavaScript no momento em que o utilizador tenta avançar de passo. A função val() é um auxiliar reutilizável que verifica um campo e mostra ou esconde a mensagem de erro.

JavaScript — Função de validação reutilizável
// val(id, errId, função_de_validação)
function val(id, errId, check) {
  const el = document.getElementById(id);
  const ok = check(el.value);       // chama a função de teste

  // Adiciona/remove classe .error no campo
  el.classList.toggle('error', !ok);

  // Mostra/esconde a mensagem de erro
  document.getElementById(errId)
    .classList.toggle('show', !ok);

  return ok;
}

// Validação do Passo 1
function validateStep1() {
  let ok = true;
  ok &= val('nome',    'err-nome',
    v => v.trim().length > 1);
  ok &= val('email',   'err-email',
    v => /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(v));
  ok &= val('telemovel','err-telemovel',
    v => v.trim().length >= 9);
  return !!ok;
}

A função de resumo do passo 3 lê os valores dos campos e injeta-os nos elementos do resumo:

JavaScript — Preencher o resumo
function preencherResumo() {
  const nome = document.getElementById('nome').value
             + ' '
             + document.getElementById('apelido').value;

  const cursoEl   = document.getElementById('curso');
  const cursoNome = cursoEl.options[cursoEl.selectedIndex]?.text;

  const turno = document
    .querySelector('input[name="turno"]:checked')?.value || '—';

  const nasc = document.getElementById('nascimento').value;

  // Injeta nos elementos do resumo
  document.getElementById('sum-nome').textContent   = nome;
  document.getElementById('sum-email').textContent  = email;
  document.getElementById('sum-nasc').textContent   =
    nasc ? new Date(nasc).toLocaleDateString('pt-PT') : '—';
  document.getElementById('sum-curso').textContent  = cursoNome;
  document.getElementById('sum-turno').textContent  = turno;
}
10

Ecrã de Sucesso e Referência

Após a submissão, o formulário é ocultado e aparece um ecrã de confirmação com animação CSS e um número de referência gerado aleatoriamente.

JavaScript — Gerar referência e mostrar sucesso
function submeterInscricao() {
  // Verifica os termos e condições
  if (!document.getElementById('chk-termos').checked) {
    document.getElementById('err-termos').classList.add('show');
    return;
  }

  // Gerar referência única: TA-2025-XXXXX
  const ref = 'TA-'
    + new Date().getFullYear()
    + '-'
    + String(Math.floor(Math.random() * 90000) + 10000);

  // Preencher o ecrã de sucesso
  document.getElementById('suc-ref').textContent  = ref;
  document.getElementById('suc-data').textContent =
    new Date().toLocaleString('pt-PT');

  // Esconder o formulário e mostrar o ecrã final
  document.getElementById('main-form').classList.add('hidden');
  document.getElementById('success-screen').style.display = 'block';
}

O ícone de visto verde tem uma animação de entrada definida em CSS:

CSS — Animação do ecrã de sucesso
.check-anim {
  animation: popIn .4s cubic-bezier(.34, 1.56, .64, 1);
}

@keyframes popIn {
  from { transform: scale(.5); opacity: 0; }
  to   { transform: scale(1);  opacity: 1; }
}
/* cubic-bezier(.34,1.56,.64,1) cria um efeito "elástico"
   — o círculo ultrapassa ligeiramente o tamanho final
   antes de estabilizar */
11

Design Responsivo

O formulário adapta-se a ecrãs pequenos com uma media query que altera o layout de 2 colunas para 1 coluna em dispositivos com largura inferior a 560px.

CSS — Media query para mobile
@media (max-width: 560px) {
  /* Grelha de 2 colunas passa para 1 */
  .form-row {
    grid-template-columns: 1fr;
  }

  /* Menos padding nos ecrãs pequenos */
  .card-body {
    padding: 1.25rem;
  }

  /* Botões a 100% de largura */
  .btn-row {
    justify-content: stretch;
  }
  .btn {
    flex: 1;
    justify-content: center;
  }
}
✅ O título principal usa clamp() para escalar automaticamente o tamanho da fonte entre um mínimo e um máximo conforme a largura do ecrã: font-size: clamp(22px, 4vw, 32px).
⚠️ Este formulário é um protótipo frontend. Numa aplicação real, os dados seriam enviados para um servidor PHP com action="processar.php" method="POST", validados novamente no servidor e guardados numa base de dados MySQL com prepared statements para evitar SQL Injection.
