Módulo 1

Introdução à Programação Web

Conceitos base da arquitetura web, protocolos e tecnologias envolvidas.

A programação web envolve a criação de aplicações que correm num browser ou num servidor web. Combina três camadas: estrutura (HTML), apresentação (CSS) e comportamento (JavaScript/PHP).

Arquitetura Cliente-Servidor

🖥 Servidor

Processa pedidos HTTP, acede a bases de dados e gera respostas. Exemplos: Apache, Nginx, Node.js. O código nunca é visto pelo utilizador.

🌐 Browser (Cliente)

Recebe HTML/CSS/JS, renderiza a página e interage com o utilizador. Chrome, Firefox, Safari são exemplos de clientes web.

O ciclo HTTP

1. Utilizador escreve URL no browser
2. Browser envia HTTP GET ao servidor
3. Servidor processa o pedido (PHP, Node.js, etc.)
4. Servidor devolve resposta 200 OK + HTML/JSON
5. Browser renderiza a página ao utilizador
O protocolo HTTP é stateless — cada pedido é independente. Para manter estado (ex: sessão de login) usa-se sessions, cookies ou tokens JWT.

Módulo 2

Scripting: Servidor vs Cliente

Diferenças entre código executado no servidor (PHP) e no browser (JavaScript).

▲ Servidor

PHP / Node.js / Python

Executa no servidor. O utilizador nunca vê o código. Pode aceder a ficheiros, bases de dados e APIs privadas.

◆ Cliente

JavaScript no browser

Executa no browser do utilizador. Visível em "Ver código fonte". Sem acesso direto a BD.

Exemplo PHP (Servidor)

PHP
// ficheiro: pagina.php — o utilizador nunca vê este código
<?php
  $nome = "Maria";
  $hora = date("H");
  if ($hora < 12) {
    $saudacao = "Bom dia";
  } else {
    $saudacao = "Boa tarde";
  }
?>
<p><?= $saudacao ?>, <?= $nome ?>!</p>

// O browser recebe APENAS: <p>Bom dia, Maria!</p>

Exemplo JavaScript (Cliente)

JavaScript
// Executa diretamente no browser — visível no DevTools
const nome = "Maria";
const hora = new Date().getHours();
const saudacao = hora < 12 ? "Bom dia" : "Boa tarde";
document.getElementById("msg").textContent = `${saudacao}, ${nome}!`;

// Fetch com async/await — pedido GET
async function carregarProdutos() {
  try {
    const resposta = await fetch('api/produtos.php');
    if (!resposta.ok) throw new Error('Erro HTTP: ' + resposta.status);

    dados.forEach(item => {
      const li = document.createElement('li');
      li.textContent = item.nome;
      lista.appendChild(li);
    });
  } catch (erro) {
    console.error('Falha:', erro);
  }
}

Módulo 4e

Scope e Tempo de Vida de Variáveis

Onde e por quanto tempo as variáveis existem em memória.

Scope em PHP

PHP
$global = "sou global";

function minhaFuncao() {
    // $global NÃO está acessível aqui automaticamente
    echo $global; // ⚠ Aviso: Undefined variable

    // Para aceder: declarar com global
    global $global;
    echo $global; // "sou global"

    // Variável local — só existe dentro desta função
    $local = "apenas aqui";

    // Variável estática — persiste entre chamadas
    static $contador = 0;
    $contador++;
    echo $contador; // 1, 2, 3... em chamadas sucessivas
}
minhaFuncao();
minhaFuncao(); // contador será 2

Scope em JavaScript

JavaScript
// var — scope de função (comportamento antigo)
function exemploVar() {
  var x = 1;
  if (true) {
    var x = 2; // MESMA variável! (var ignora blocos)
  }
  console.log(x); // 2
}

// let/const — scope de bloco {} (recomendado)
let y = 10;
if (true) {
  let y = 20; // variável diferente, dentro do bloco
  console.log(y); // 20
}
console.log(y); // 10 — a exterior mantém-se

// Closure — função "fecha" sobre variáveis do scope exterior
function criarContador() {
  let n = 0; // n persiste via closure
  return () => ++n;
}
const inc = criarContador();
inc(); // 1
inc(); // 2
inc(); // 3
Usar var pode causar bugs difíceis de detetar devido ao hoisting e scope de função. Prefira sempre let e const.

Eventos Globais e Indiretos

addEventListener, eventos do DOM, da janela e delegação de eventos.

Eventos diretos (no elemento)

JavaScript
const btn = document.getElementById('meuBotao');

// Adicionar listener
btn.addEventListener('click', (e) => {
  console.log('Clicado!', e.target);
  e.preventDefault(); // previne comportamento padrão
});

btn.addEventListener('mouseover', () => {
  btn.style.background = 'lightblue';
});
btn.addEventListener('mouseout', () => {
  btn.style.background = '';
});

// Remover listener (necessita referência à função)
function handler(e) { console.log(e.type); }
btn.addEventListener('click', handler);
btn.removeEventListener('click', handler);

Eventos globais (window / document)

// DOM pronto — sem esperar imagens e recursos
document.addEventListener('DOMContentLoaded', () => {
  inicializar();
});

// Página completamente carregada (imagens incluídas)
window.addEventListener('load', () => {
  console.log('Tudo carregado');
});

// Redimensionar janela
window.addEventListener('resize', () => {
  console.log(window.innerWidth, window.innerHeight);
});

// Tecla premida
document.addEventListener('keydown', (e) => {
  if (e.key === 'Escape') fecharModal();
  if (e.ctrlKey && e.key === 's') { e.preventDefault(); guardar(); }
});

// Antes de sair da página
window.addEventListener('beforeunload', (e) => {
  e.returnValue = 'Tens alterações não guardadas!';
});

Delegação de eventos (indiretos)

// Em vez de adicionar listener a cada item da lista,
// adicionar ao pai — funciona para elementos futuros!
const lista = document.getElementById('minhaLista');

lista.addEventListener('click', (e) => {
  // e.target é o elemento realmente clicado
  if (e.target.matches('li.item')) {
    e.target.classList.toggle('selecionado');
    console.log('Item:', e.target.textContent);
  }
  if (e.target.matches('.btn-apagar')) {
    e.target.closest('li').remove();
  }
});
A delegação de eventos é mais eficiente do que adicionar um listener a cada elemento — especialmente em listas longas ou elementos criados dinamicamente.

Base de Dados

CRUD com PHP e MySQL

Create, Read, Update, Delete — as quatro operações fundamentais sobre dados.

▲ Servidor 🗄 MySQL / PDO
PHP — PDO
// Ligação com PDO — suporta MySQL, PostgreSQL, SQLite...
$host = 'localhost';
$bd   = 'minha_bd';
$user = 'root';
$pass = 'senha';

try {
    $pdo = new PDO(
        "mysql:host=$host;dbname=$bd;charset=utf8mb4",
        $user, $pass
    );
    $pdo->setAttribute(PDO::ATTR_ERRMODE,
                     PDO::ERRMODE_EXCEPTION);
    echo "Ligação bem-sucedida!";
} catch (PDOException $e) {
    die("Erro de ligação: " . $e->getMessage());
}

// Criar tabela exemplo
$pdo->exec("CREATE TABLE IF NOT EXISTS utilizadores (
  id    INT AUTO_INCREMENT PRIMARY KEY,
  nome  VARCHAR(100) NOT NULL,
  email VARCHAR(150) UNIQUE NOT NULL,
  idade INT
)");
PDO é preferível a mysqli porque suporta múltiplos motores de BD com a mesma sintaxe e facilita a migração de base de dados.
PHP — INSERT
// CREATE — Inserir dados com prepared statement
$sql = "INSERT INTO utilizadores (nome, email, idade)
         VALUES (:nome, :email, :idade)";

$stmt = $pdo->prepare($sql);
$stmt->execute([
    ':nome'  => $_POST['nome'],
    ':email' => $_POST['email'],
    ':idade' => $_POST['idade']
]);

$idNovo = $pdo->lastInsertId();
echo "Utilizador inserido com ID: $idNovo";
Nunca concatenes dados do utilizador diretamente no SQL! "...WHERE nome='$_POST[nome]'" é vulnerável a SQL Injection. Usa sempre prepared statements com :parametro.
PHP — SELECT
// READ — Todos os registos
$stmt = $pdo->query(
    "SELECT * FROM utilizadores ORDER BY nome ASC"
);
$utilizadores = $stmt->fetchAll(PDO::FETCH_ASSOC);

foreach ($utilizadores as $u) {
    echo $u['nome'] . ' — ' . $u['email'];
}

// Com filtro por ID (prepared statement)
$stmt = $pdo->prepare(
    "SELECT * FROM utilizadores WHERE id = :id"
);
$stmt->execute([':id' => $_GET['id']]);
$registo = $stmt->fetch(PDO::FETCH_ASSOC);

// Pesquisa por nome (LIKE)
$stmt = $pdo->prepare(
    "SELECT * FROM utilizadores WHERE nome LIKE :termo"
);
$stmt->execute([':termo' => '%' . $_GET['q'] . '%']);
PHP — UPDATE
// UPDATE — Atualizar um registo existente
$sql = "UPDATE utilizadores
         SET nome  = :nome,
             email = :email,
             idade = :idade
         WHERE id  = :id";

$stmt = $pdo->prepare($sql);
$stmt->execute([
    ':nome'  => $_POST['nome'],
    ':email' => $_POST['email'],
    ':idade' => $_POST['idade'],
    ':id'    => $_POST['id']
]);

$alterados = $stmt->rowCount();
echo "$alterados registo(s) atualizado(s)";
PHP — DELETE
// DELETE — Eliminar um registo
$stmt = $pdo->prepare(
    "DELETE FROM utilizadores WHERE id = :id"
);
$stmt->execute([':id' => $_POST['id']]);

echo $stmt->rowCount() > 0
    ? "Registo eliminado com sucesso."
    : "Nenhum registo encontrado.";

// Alternativa: soft delete (não elimina fisicamente)
$pdo->prepare(
    "UPDATE utilizadores SET ativo = 0 WHERE id = :id"
)->execute([':id' => $_POST['id']]);
Considera usar soft delete (marcar como inativo) em vez de apagar fisicamente — permite recuperar dados acidentalmente eliminados.
