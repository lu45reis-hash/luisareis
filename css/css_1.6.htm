Definindo estilos

Externo · Interno · Inline

Hoje em dia, definimos estilos de três formas: CSS externo (ficheiro .css), CSS interno (tag <style>) e CSS inline (atributo style=""). Em sites com várias páginas, a melhor opção quase sempre é o CSS externo, porque facilita a manutenção e mantém consistência visual.

1) CSS externo (recomendado)

Criamos um ficheiro .css e ligamo-lo no <head>: 

<link rel="stylesheet" href="estilos/cursos.css">
- Ideal para sites: reutilizas o mesmo CSS em várias páginas e alteras tudo num só ficheiro.

2) CSS interno (para páginas isoladas ou testes)

Coloca-se dentro de <style> no <head>. Útil quando o estilo é específico daquela página. 

<head>
  <style>
    h1 { color: red; font-style: italic; }
    blockquote { text-transform: uppercase; }
  </style>
</head>

<body>
  <h1>Título do exemplo</h1>
  <blockquote>Citação de exemplo</blockquote>
</body>

3) CSS inline (evitar na maioria dos casos)

O estilo é aplicado diretamente no elemento. É prático, mas torna o código difícil de manter. Usa apenas em casos pontuais. 

<p style="color:#0ea5e9; font-weight:700;">
  Texto com estilo inline
</p>

Boas práticas atuais

  • Preferir CSS externo para projetos com várias páginas.
  • Usar classes para reutilizar estilos (ex.: .card, .btn).
  • Evitar repetir o mesmo estilo inline muitas vezes.
