Estilos e HTML

CSS (Cascading Style Sheets)

O HTML define a estrutura e o significado do conteúdo (títulos, parágrafos, listas, imagens). O CSS define a apresentação (cores, tipografia, espaçamentos, layout e responsividade).

- Boas práticas atuais: mantém o HTML “limpo” (sem estilos no meio do conteúdo) e usa CSS para controlar o visual. Isso facilita manutenção, acessibilidade e consistência do site.

3 formas de aplicar CSS

1) CSS externo (recomendado)

<!-- no <head> -->
<link rel="stylesheet" href="estilos.css">

Ideal para sites com várias páginas (reutilização e organização).

2) CSS interno (no próprio ficheiro)

<style>
  body { font-family: system-ui; }
  h1 { color: #1d4ed8; }
</style>

Útil para exemplos rápidos ou páginas muito pequenas.

3) CSS inline (evitar, regra geral)

<p style="color:#ef4444; font-weight:700;">
  Texto a vermelho
</p>

Dificulta manutenção. Usa apenas em casos pontuais.

Separação de responsabilidades

<!-- HTML: estrutura -->
<button class="btn">Enviar</button>

/* CSS: visual */
.btn { padding: 10px 14px; border-radius: 12px; }

HTML com classes + CSS a definir estilos (padrão moderno).

O que significa “cascata”

Se existirem várias regras para o mesmo elemento, o browser resolve conflitos por: origem da regra (user-agent, autor), especificidade (id > class > tag) e ordem (a última regra pode prevalecer).

Dica rápida: evita abusar de !important. Em vez disso, melhora a estrutura do CSS e usa classes de forma consistente.
