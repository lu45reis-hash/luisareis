Formatação de Caracteres

bloco
  |__ Semânticos (recomendados): 
      strong, em, 
      cite, code, 
      samp, kbd, 
      var, dfn
  |__ Apenas apresentação: 
      b, i, 
      u, s, 
      sub, sup
  |__ span (genérico)

Em HTML5, a formatação de caracteres deve priorizar o significado em vez da aparência visual. Por isso, os elementos semânticos são fortemente recomendados.

Regra atual: Use sempre que possível os elementos semânticos (<strong>, <em>, <code>, etc.). Eles transmitem significado aos leitores de ecrã, motores de busca e ferramentas de acessibilidade.

Elementos Semânticos (Recomendados)

<strong>Texto com forte importância</strong>
<em>Texto enfatizado</em>
<cite>Referência a uma obra ou citação</cite>
<code>Código fonte</code>
<samp>Saída de um programa</samp>
<kbd>Entrada do utilizador/teclado</kbd>
<var>Nome de variável</var>
<dfn>Definição de um termo</dfn>

Este texto tem forte importância
Este texto está enfatizado
Referência bibliográfica
console.log("Olá Mundo");
Saída do programa
Ctrl + C
x = 42

Elementos de Apresentação (Apenas visual)

<b>Negrito visual</b>
<i>Itálico visual</i>
<u>Sublinhado visual</u>
<s>Texto riscado</s>
<sub>Subscrito</sub>
<sup>Sobrescrito</sup>

Texto em negrito (apenas visual)
Texto em itálico (apenas visual)
Texto sublinhado (apenas visual)
Texto riscado
H₂O  E = mc²

<span> ... </span>

Elemento genérico inline sem significado semântico. Deve ser usado apenas quando nenhum outro elemento semântico se adequa, combinado com CSS.

<span style="color: #e74c3c; font-weight: 700; background-color: #fff3cd;">Trecho destacado</span>
Resumo atual (HTML5):
• Prefira <strong> em vez de <b>
• Prefira <em> em vez de <i>
• Use <code>, <kbd>, <samp> para código e entradas
• <span> só quando realmente necessário + CSS
