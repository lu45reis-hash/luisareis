⚠️ Nota importante • HTML5

No HTML antigo existia o sistema de frames (com <frameset>) para dividir a janela do browser em várias áreas (menu, topo, conteúdo, etc.). Em HTML5, frames são obsoletos e não devem ser usados em sites atuais.

Porquê evitar frames? Trazem problemas de acessibilidade, navegação/URLs, SEO, histórico do browser e adaptação a ecrãs pequenos.

O que se usa hoje no lugar de frames?

1) Layout com CSS (o mais comum)

Para criar um site com menu fixo + conteúdo ou topo + área principal, usa-se CSS Grid ou Flexbox. 

/* Exemplo: layout com menu + conteúdo */
.layout{
  display: grid;
  grid-template-columns: 240px 1fr;
  min-height: 100vh;
}

aside{ padding: 16px; }
main{ padding: 16px; }

2) <iframe> (para incorporar uma página)

Quando precisas de mostrar uma página externa/embebida (ex.: mapa, vídeo, documento), podes usar <iframe>, com moderação. 

<iframe
  src="https://example.com"
  title="Página incorporada"
  width="100%"
  height="420"
  loading="lazy"
  referrerpolicy="no-referrer"
></iframe>
Resumo: - Para “dividir” a página em áreas CSS (Grid/Flex) - Para embutir conteúdo externo <iframe>
