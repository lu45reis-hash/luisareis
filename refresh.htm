Um redirecionamento serve para levar o utilizador automaticamente para outra página. Em HTML existe o método do meta refresh, mas hoje em dia a melhor prática é fazer o redirecionamento no servidor (quando possível).

- Nota importante: meta refresh pode ser mau para acessibilidade e SEO. Usa apenas em casos simples (ex.: páginas antigas, mensagens temporárias ou demos).

1) Meta Refresh (HTML)

Esta meta tag define um tempo (em segundos) e a URL de destino. Exemplo: redireciona após 5 segundos.

Código

<!-- no <head> -->
<meta http-equiv="refresh" content="5; url=../index.html">
Como ler:
content="5; url=..." espera 5 segundos e depois abre a URL.

Exemplo de página (com mensagem)

<!doctype html>
<html lang="pt-PT">
<head>
  <meta charset="utf-8">
  <meta http-equiv="refresh" content="5; url=../index.html">
  <title>A redirecionar…</title>
</head>
<body>
  <p><strong>Dentro de 5 segundos voltas à página anterior.</strong></p>
  <p>Se não acontecer, clica <a href="../index.html">aqui</a>.</p>
</body>
</html>
- Boa prática: inclui sempre um link manual (“se não funcionar, clica aqui”).

2) Alternativa recomendada: redirecionar no servidor

Quando tens controlo do servidor, é preferível um redirecionamento HTTP:

  • 301 (permanente) — quando uma página mudou definitivamente
  • 302 (temporário) — quando a mudança é provisória

(Aqui no manual não vamos configurar servidor, mas é importante saber que é a opção mais “atual”.)
